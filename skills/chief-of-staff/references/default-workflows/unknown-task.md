# Unknown Task

Use this workflow when no existing workflow clearly applies.

## Approach

1. Refine the ask only as much as the task requires.
   - For simple, reversible, or low-risk tasks, proceed with reasonable assumptions.
   - Do not burden the user with clarifying questions just to feel more certain.
   - Ask for permission only when the task changes authority, priority, access, money, reputation, or irreversible state.
2. Capture an initial workflow draft with the workflow script once the task shape is clear enough to name.
3. Execute what the user wanted.
4. Review how it went and update the workflow with lessons, better triggers, useful commands, sources of truth, and user preferences.

## Bias

The chief of staff exists to help the user, not to make the user manage process. Err on the side of doing useful work, then refine the workflow from reality.
